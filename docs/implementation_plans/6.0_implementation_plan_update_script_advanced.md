# Advanced Copilot Forge Update Script

The suggestions from the other LLM outline a significantly more robust, "enterprise-grade" update tool compared to the simple zip-download mechanism we just built. This new approach shifts the paradigm: instead of the script downloading the zip, the script lives in the user's cloned toolkit repository and pushes updates *out* to target projects.

## Proposed Changes

We will completely rewrite `scripts/update.ps1` to incorporate all the suggested features:

### 1. Parameter Expansion
- `[string[]]$TargetDir`: Support updating multiple repositories at once (e.g., in a monorepo).
- `[switch]$DryRun`: Preview mode to see what would change without modifying files.
- `[switch]$SkipGitPull`: Bypass pulling the toolkit repo if offline or already synced.
- `[switch]$Force`: Skip interactive confirmation prompts for CI/CD automation.

### 2. Toolkit Auto-Sync
If `-SkipGitPull` is not passed, the script will execute `git pull` from `$PSScriptRoot/..` to ensure the local toolkit clone has the latest prompts/agents before deploying them to the targets.

### 3. Smart Sync & Protection
Instead of blind recursive copies:
- Use `Get-FileHash` to compare MD5 hashes. Only copy files that have genuinely changed, preventing git history noise from timestamps.
- **Protect `copilot-instructions.md`**: Skip this file with a warning if the hash differs from the toolkit version, preventing the user's custom ethos from being overwritten.

### 4. Template Drift Health Report
For the `templates/` directory, we will calculate the drift instead of just doing an ignorant merge:
- Use `git diff --no-index --numstat` (if `git` is available) or a native PowerShell comparison to calculate added/removed lines.
- Classify each template as:
  - ✅ **Identical**: Hashes match.
  - 🆕 **New**: Doesn't exist in target; copy it over.
  - ⚠️ **Drifted**: Exists in both but hashes differ. Leave untouched and report the diff size.
  - 🔒 **Custom**: Exists only in target; ignore.
- Print a formatted table showing the Health Report for the drifted templates.

### 5. Version Stamping
Extract the version from `README.md` changelog (e.g., `v2.8.0`) or maintain a `VERSION` file, and write it to `$TargetDir/.forge/.forge-version` to track what version the project is currently running.

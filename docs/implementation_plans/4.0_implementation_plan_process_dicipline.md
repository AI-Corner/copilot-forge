# Feature 16: Process Discipline Guides

Currently, Copilot Forge dictates *workflow* (what to do and when), but it lacks explicit rules around *behavioral discipline* (e.g., when to stop, when to push back, how to edit surgically, how to avoid hallucinations). This feature introduces explicit process guides.

## Proposed Changes

### 1. New Process Guide Templates
Create a new set of templates inside `templates/process/` representing the tiered discipline guides:

- **Iron Laws (`iron-laws-template.md`)**: The 7 non-negotiable rules (e.g., no proceeding without ACs, no silent overwrites, no --no-verify). 🔴 **IRON LAW** severity.
- **Grounding (`grounding-template.md`)**: Verify references exist before using them.
- **Self-Validation (`self-validation-template.md`)**: Don't count your own claims as evidence.
- **Pushback (`pushback-template.md`)**: Disagree when the user is wrong or the premise is flawed.
- **Escalation (`escalation-template.md`)**: When to stop trying and ask the user for help.
- **Context Budget (`context-budget-template.md`)**: Read enough and no more.
- **Scoping (`scoping-template.md`)**: One concern per commit, max 500 lines.
- **Surgical Edits (`surgical-edits-template.md`)**: Touch only what serves the task.
- **Subagent Trust (`subagent-trust-template.md`)**: Verify subagent work by reading the diff.

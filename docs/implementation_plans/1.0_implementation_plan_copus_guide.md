# Feature 14: Corpus/Guide Split (Context Window Optimization)

This plan implements Feature 14 from the `ROADMAP.md`. We will split the monolithic `.forge/context/` files into two layers: short, directive `rules/` (always loaded) and long-form `corpus/` (loaded on demand). This minimizes token usage and improves agent focus.

## Dependencies

- **Prerequisites**: None. Feature 14 has no dependencies.
- **Downstream Dependencies**: Feature 15 (Learning Flywheel) and Feature 19 (Drift Sensor) depend heavily on this feature being completed first.

### File Migration
We will move the detailed context files to the `corpus/` directory:
- `architecture.md` -> `corpus/architecture.md`
- `conventions.md` -> `corpus/conventions.md`
- `variables.md` -> `corpus/variables.md`

`project-overview.md` will remain in `.forge/context/` as it is short and always loaded.

### Rule Extraction
We will create distilled, directive rule files in the new `rules/` directory:
- `rules/architecture.rules.md`: Key structural constraints extracted from `architecture.md`.
- `rules/conventions.rules.md`: Key coding and formatting rules extracted from `conventions.md`.
- `rules/security.rules.md`: Rules derived from `security_scan.prompt.md` (e.g., hardcoded secret checks).
- `rules/deployment.rules.md`: Rules derived from `canary.prompt.md` (e.g., zero-traffic deployments, health checks).

### Prompt Updates
We will update all the main prompts and agent checklists in `.github/prompts/` to adopt the new structure.

We will inject the following preamble into the relevant prompts:
```markdown
## Context Loading Rule
1. ALWAYS read `.forge/context/rules/` files — these are your constraints.
2. Read `.forge/context/corpus/` files ONLY when a rule references them
   or when the rule alone is ambiguous for the current situation.
```

Files to be updated include (but are not limited to):
- `spec.prompt.md`, `architect.prompt.md`, `review.prompt.md`, `reflect.prompt.md`
- `wrapup.prompt.md`, `validate.prompt.md`, `tdd.prompt.md`, `sprint.prompt.md`
- `query.prompt.md`, `token-estimate.prompt.md`, `proceed.prompt.md`

We will replace all direct references to `.forge/context/architecture.md` and `.forge/context/conventions.md` with the new pattern.

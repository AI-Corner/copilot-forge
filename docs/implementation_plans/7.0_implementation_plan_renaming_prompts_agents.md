# Feature: Namespace All Forge Prompts to Avoid VS Code Popup Clutter

## Problem
When typing `#` in VS Code Copilot Chat, all `.prompt.md` files from every repo in the workspace show up in the autocomplete popup. Forge's current prompt names are generic (`spec`, `review`, `init`, `analyze`, etc.) and can collide with prompts from other tools, other repos, or even VS Code's own suggestions. This creates a cluttered, confusing popup — especially for users with multiple repos open.

## Proposed Solution: Add a `forge-` Prefix to All Prompts

Rename every top-level prompt from `<name>.prompt.md` to `forge-<name>.prompt.md`. This namespaces them so typing `#forge-` in the popup immediately filters to only Forge commands.

## Open Questions

> [!IMPORTANT]
> **Sub-agents too?** The `agents/` subdirectory has 23 prompt files (e.g. `reflector.prompt.md`, `architecture-reviewer.prompt.md`). These are internal — invoked by parent prompts, not typed by the user. **Should we also prefix these with `forge-`?** They do appear in the popup today. I recommend **yes** for consistency, but they could also be left as-is since users never invoke them manually.

> [!IMPORTANT]
> **Underscore normalization?** `security_scan` and `issue_epic_creation` use underscores while everything else uses hyphens. Should we normalize these to hyphens during the rename (e.g. `forge-security-scan`, `forge-issue-epic-creation`)?

> [!IMPORTANT]
> **`forge-admin` already has the prefix.** It stays as-is — no change needed.

## Blast Radius — Files Requiring Cross-Reference Updates

This is a **breaking rename** — every internal `#spec`, `#review`, etc. reference must be updated. The affected files are:

### Prompt Files (26 top-level + 23 agents)
Every prompt file internally references sibling prompts (e.g. `proceed.prompt.md` calls `#spec`, `#architect`, `#tdd`, `#reflect`, `#review`, `#wrapup`). All these references must be updated.

### Documentation (4 files)
- **`README.md`** — changelog entries, workflow examples, prompt names throughout
- **`FAQ.md`** — FAQ #26, #27, and many earlier entries reference `#spec`, `#review`, etc.
- **`PROMPT_USAGE.md`** — the entire reference guide (~571 lines) with every prompt name
- **`ROADMAP.md`** — references to prompt names in feature descriptions

### Templates (13 files)
- All files in `templates/` that reference prompt names in instructions or examples

### Scripts (1 file)
- **`scripts/install.ps1`** — line 54 references `#init`

### Config / Instructions
- **`.github/copilot-instructions.md`** — may reference prompt names

### Graph Data
- **`docs/graph/data.js`** — node/edge IDs referencing prompt names

## Proposed Changes

### Phase 1: Rename Prompt Files
- `git mv` all 26 top-level prompt files to `forge-<name>.prompt.md`
- `git mv` all 23 agent prompt files to `forge-<name>.prompt.md` (if approved)
- Normalize underscores to hyphens

### Phase 2: Update Internal Cross-References
- Search-and-replace `#<old-name>` → `#forge-<new-name>` in all `.prompt.md` files
- Update any `applyTo` or `description` YAML frontmatter if it references prompt names

### Phase 3: Update Documentation
- Update `README.md`, `FAQ.md`, `PROMPT_USAGE.md`, `ROADMAP.md`
- Update `scripts/install.ps1`
- Update `.github/copilot-instructions.md`
- Update `docs/graph/data.js`
- Update `templates/*.md`

### Phase 4: Update Install Script
- Update line 54 in `install.ps1` to reference `#forge-init`

## Verification Plan

### Automated
- `grep -r "#spec\b" .github/prompts/` should return zero hits (confirming all old refs are gone)
- Repeat for every renamed prompt
- `git diff --stat` to confirm the rename scope

### Manual
- Open target repo in VS Code, type `#forge-` in Copilot Chat and verify all prompts appear with prefix
- Run `#forge-init` to confirm it still works end-to-end

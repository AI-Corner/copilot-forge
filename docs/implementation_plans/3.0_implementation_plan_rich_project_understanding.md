# Feature 20: Codebase State Document (Rich Project Understanding)

Currently, the Forge context captures intent (`architecture.md`, `conventions.md`) but doesn't track the **empirical state** of the project—like which frameworks are actually installed, the current health score, and the accumulated technical debt. This feature addresses that gap by introducing a trio of empirical context documents.

## Proposed Changes

### 1. New Context Documents

Enhance `.forge/context/` to include these three files (added to `.gitignore` exclusions if necessary, though they should be tracked in git):
- `codebase-state.md` — Real, empirical state (stack, framework versions, tool commands, CI platform).
- `quality-radar.md` — 5-dimension scorecard (A-F grades) representing project health.
- `code-debt.md` — Tracked ledger of technical debt items, allowing the AI to know what needs fixing without rescanning.

### 2. Update `#init` Prompt

#### [MODIFY] [init.prompt.md](file:///d:/2026/ai-corner/copilot-forge/.github/prompts/init.prompt.md)
Update `#init` to automatically generate `codebase-state.md` during the initial setup by scanning `package.json`, `pom.xml`, or other build files. It will also scaffold empty `quality-radar.md` and `code-debt.md` files.

### 3. Update `#analyze` Prompt

#### [MODIFY] [analyze.prompt.md](file:///d:/2026/ai-corner/copilot-forge/.github/prompts/analyze.prompt.md)
Modify the `#analyze` output step (Step 3/4) to actually write its findings to the file system instead of just outputting them in the chat window:
- Write the final scorecard to `.forge/context/quality-radar.md`.
- Append technical debt items and FIXME/TODO counts to `.forge/context/code-debt.md`.
- Update tool versions in `.forge/context/codebase-state.md` if they changed.

### 4. Update `#review` Prompt

#### [MODIFY] [review.prompt.md](file:///d:/2026/ai-corner/copilot-forge/.github/prompts/review.prompt.md)
When `#review` identifies "Minor" issues or "Improvement Opportunities" that the user chooses to defer rather than fix immediately, `#review` should append these items to `.forge/context/code-debt.md` automatically so they aren't lost.

### 5. Documentation Sync

- Update `README.md` with a `v2.6.0` changelog.
- Update `ROADMAP.md` to mark Feature 20 as ✅ Implemented.
- Update `PROMPT_USAGE.md` to reflect that `#init`, `#analyze`, and `#review` now interact with the new codebase state documents.

## Open Questions

> [!IMPORTANT]
> Should `code-debt.md` use the standard `inbox-template.md` routing, or should it be a simple standalone markdown list? Given it tracks technical debt rather than process knowledge, I recommend a standalone `code-debt.md` file formatted as a ledger table so it's easy to read and update.

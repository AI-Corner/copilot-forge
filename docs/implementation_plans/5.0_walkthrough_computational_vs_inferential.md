# Walkthrough: Computational vs. Inferential Sensor Split (Feature 17)

**Feature 17 (Computational vs. Inferential Sensor Split)** has been successfully implemented and shipped in **v2.8.0**!

## What We Built

Prior to this feature, our validation phases combined deterministic shell commands (like running a linter) with expensive, time-consuming LLM reviews. This meant an LLM might spend 40 seconds reviewing a file for architectural purity, only for the build script to subsequently fail because of a missing semicolon.

We have now explicitly bifurcated the verification loop into two rigid stages:

### 1. Computational Sensors (Part 1)
Located in `.github/prompts/agents/computational/`, these are fast, deterministic shell commands that return binary PASS/FAIL results. We created several new ones:
- `lint-gate.prompt.md`
- `typecheck-gate.prompt.md`
- `build-gate.prompt.md`
- `test-gate.prompt.md`
- `secret-scan.prompt.md`
- `vuln-scan.prompt.md`

**The Rule**: If *any* of these fail, the pipeline immediately halts Phase 5 verification and forces the agent to fix the code and re-run the computational sensors. **It will not proceed to Part 2 until Part 1 is 100% clean.**

### 2. Inferential Sensors (Part 2)
Located in `.github/prompts/agents/inferential/`, these are the LLM-driven heuristics. We moved all our existing auditors here:
- `correctness-reviewer.prompt.md`
- `quality-reviewer.prompt.md`
- `architecture-reviewer.prompt.md`
- `test-auditor.prompt.md`
- `security-auditor.prompt.md`
- `code-quality-auditor.prompt.md`
- `convention-auditor.prompt.md`

**New Addition**: We also created `spec-adherence.prompt.md` which is dedicated exclusively to reading the diff and verifying that every single Acceptance Criterion was met, and that no unrequested features (scope creep) were added.

### 3. Core Prompts & Dependency Graph
- Updated **`proceed.prompt.md`** Phase 5 to execute Part 1 (Computational) first, and Part 2 (Inferential) second.
- Updated **`review.prompt.md`** and **`analyze.prompt.md`** to point to the new `inferential/` subdirectories.
- Automatically re-generated the interactive **`forge-dependency-graph.md`** to visualize the new agent paths!

### 4. Documentation
- Updated **`README.md`** with the v2.8.0 changelog.
- Marked Feature 17 as ✅ Implemented in **`ROADMAP.md`**.
